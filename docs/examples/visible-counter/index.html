Visible Tag Counter For Hiring Skills

Recruiters need a visible count while selecting the few skills that matter most for a role.

Scenario

User goal: choose up to four role skills and understand when the limit has been reached.

Try it

  • Add tags until the counter reaches four of four.
  • Try typing another tag after the limit is reached.
  • Remove one tag and confirm the counter updates.
  • Submit the form to inspect the synchronized value.

Developer notes

The plugin has a hidden counter for assistive tech when maxTags is finite. This demo adds a visible counter beside it. 

const instance = createTagInput(source, {
  maxTags: 4,
  lowercase: true
});

How to build it

Add a visible live output, then refresh its text whenever the plugin reports that a tag was added or removed.

HTML

<textarea id="skill-tags"></textarea>
<output id="tag-counter-output" aria-live="polite">
  0 of 4 skills selected.
</output>

JavaScript

const instance = createTagInput(source, { maxTags: 4 });

const updateCounter = () => {
  counter.value = `${instance.getTags().length} of 4 skills selected.`;
};

source.addEventListener("a11y-tag-input:add", updateCounter);
source.addEventListener("a11y-tag-input:remove", updateCounter);

Accessibility notes

  • The visible counter is an output with polite updates.
  • The plugin also keeps its internal counter connected to the generated input.
  • The limit state uses text and status messages, not color alone.
  • Keep visible counters close to the field they describe.

Limitations

  • This recipe demonstrates addon-style behavior around the core plugin; production forms still need target browser, assistive technology, touch, zoom, and forced-colors checks.
  • Keep labels, validation copy, and recovery instructions aligned with your own domain language before shipping.
