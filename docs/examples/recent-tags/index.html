Recent Tags For Editorial Planning

Editors often reuse a small set of topical labels. This recipe keeps recent tags close without turning the input into a custom picker.

Scenario

User goal: add frequently used editorial tags quickly, while retaining the normal tag-input editing flow.

Try it

  • Activate a recent tag button.
  • Add your own tag with Enter.
  • Remove a tag and add it again from the recent list.
  • Try adding a recent tag that is already selected.

Developer notes

Recent tags can be plain buttons that call addTag(). The plugin still handles duplicates and validation. 

button.addEventListener("click", () => {
  instance.addTag(button.dataset.tag);
});

How to build it

Render recent values as native buttons and call addTag() so the plugin applies its normal duplicate and limit checks.

HTML

<textarea id="editorial-tags"></textarea>
<div id="recent-tags" aria-label="Recent tags">
  <button type="button" data-tag="research">research</button>
</div>

JavaScript

const instance = createTagInput(source);

button.addEventListener("click", () => {
  instance.addTag(button.dataset.tag ?? "");
  instance.focus();
});

Accessibility notes

  • Recent tags are exposed as named buttons in a labelled group.
  • The core plugin announces successful adds and duplicate errors.
  • Use recent-tag storage carefully; do not store sensitive labels without a clear reason.
  • This static demo uses local in-page data only.

Limitations

  • This recipe demonstrates addon-style behavior around the core plugin; production forms still need target browser, assistive technology, touch, zoom, and forced-colors checks.
  • Keep labels, validation copy, and recovery instructions aligned with your own domain language before shipping.
