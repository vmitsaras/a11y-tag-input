Scan authored tag input markup on demand and receive structured diagnostics without changing the DOM or logging automatically. Use the results in tests, documentation previews, or your own development tooling.

Review a component example before publishing it

A design-system team is preparing a tag input example for its documentation. The first version contains several configuration mistakes that the runtime would otherwise ignore or encounter later.

Try it

  1. Activate Inspect flawed markup and confirm that four diagnostics are reported.
  2. Review each diagnostic code, severity, and recovery message in the results list.
  3. Activate Inspect corrected markup and confirm that no diagnostics are found.
  4. Use the live corrected example to add and remove tags from an initialized control.

The result count is announced through a status region. Diagnostic details remain in a normal ordered list so they can be reviewed at your own pace.

    Add the inspector to development tooling

    Import the separate development entry and run it before initializing tag inputs. The inspector returns its findings; your code decides whether to show them in test output, a documentation preview, or a development overlay. 

    import { inspectTagInputs } from "a11y-tag-input/dev";

const diagnostics = inspectTagInputs({
  root: document,
  selector: "[data-a11y-tag-input]"
});

for (const diagnostic of diagnostics) {
  reportDiagnostic({
    code: diagnostic.code,
    severity: diagnostic.severity,
    message: diagnostic.message,
    element: diagnostic.element
  });
}

    What the inspector does not establish

    The inspector checks static authored markup and selected configuration relationships. It does not reproduce the browser's complete accessible-name computation, test runtime interaction, verify screen reader output, or establish WCAG conformance. Optional WCAG references are review pointers only.

    Run it before initialization, then test the finished component with your target browsers and assistive technologies.
