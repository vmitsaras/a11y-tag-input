Addon recipe

German Messages For Every State

Product teams can replace the complete message catalog while preserving the tag input's keyboard and screen reader behavior.

Scenario

User goal: add German content topics and receive every instruction, control name, validation error, and status update in German.

Try it

  • Add gestaltung and listen for the German success message.
  • Add barrierefreiheit again to trigger the localized duplicate error.
  • Paste forschung,tests to hear a localized paste summary.
  • Reach the four-tag limit, then remove a tag using its German accessible name.

Example

Fügen Sie bis zu vier Themen hinzu. Drücken Sie die Eingabetaste, ein Komma oder Semikolon, um ein Thema hinzuzufügen.

Developer notes

Pass a partial or complete messages object at initialization. Keep tokens such as {tag}, {count}, and {max} where the translated sentence needs dynamic values. 

createTagInput(source, {
  messages: {
    removeTag: "Tag {tag} entfernen",
    addedTag: "Tag {tag} hinzugefügt.",
    counterText: "{count} von {max} Tags verwendet."
  }
});

How to build it

Provide the field in HTML, then pass translated interface and validation messages when you initialize the tag input.

HTML

<form lang="de">
  <textarea id="content-topics" name="content-topics">
barrierefreiheit,inhalte
  </textarea>
</form>

JavaScript

const source = document.querySelector("#content-topics");

createTagInput(source, {
  messages: {
    inputLabel: "Tag hinzufügen",
    removeTag: "Tag {tag} entfernen"
  }
});

Accessibility notes

  • The localized form has lang="de", so generated control names and live messages inherit German pronunciation.
  • Visible labels, hidden instructions, remove-button names, errors, counters, and status announcements use the same locale.
  • Message tokens are interpolated after localization; translators can move them to fit the target language's grammar.
  • Keyboard and focus behavior do not change when messages are replaced.
  • Review translations in context and test them with the target screen reader and browser combination.

Limitations

  • This recipe demonstrates addon-style behavior around the core plugin; production forms still need target browser, assistive technology, touch, zoom, and forced-colors checks.
  • Keep labels, validation copy, and recovery instructions aligned with your own domain language before shipping.
