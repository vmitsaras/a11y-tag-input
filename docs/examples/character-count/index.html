Character Count For Topic Tags

A product team keeps research tags short enough to scan in dashboards while the tag input enforces the same length rule.

Scenario

User goal: add concise topic tags and see how many characters remain before committing the tag.

Try it

  • Type accessibility and watch the count update.
  • Try a long tag such as internationalization.
  • Press Enter to commit a valid tag.
  • Use Backspace from an empty field to reach the final remove button.

Example

Add short topics for the research digest. Each tag can be 18 characters or fewer.

Current tag length
0 of 18 characters used.

Developer notes

This page uses the core plugin plus demo-only JavaScript that reads instance.field.value. 

const instance = createTagInput(source, {
  maxLength: 18,
  maxTags: 6
});

How to build it

Place a live output beside the field, then update it from the generated input while keeping the plugin's maximum-length validation.

HTML

<textarea id="research-tags"></textarea>
<output id="character-count-output" aria-live="polite">
  0 of 18 characters
</output>

JavaScript

const instance = createTagInput(source, { maxLength: 18 });

instance.field.addEventListener("input", () => {
  counter.value = `${instance.field.value.length} of 18 characters`;
});

Accessibility notes

  • The source textarea has a visible label and description before JavaScript runs.
  • The generated input receives the label and plugin instructions.
  • The character count is exposed through an output with polite updates.
  • The max length is enforced by the generated input and should still be tested with target browsers.

Limitations

  • This recipe demonstrates addon-style behavior around the core plugin; production forms still need target browser, assistive technology, touch, zoom, and forced-colors checks.
  • Keep labels, validation copy, and recovery instructions aligned with your own domain language before shipping.
