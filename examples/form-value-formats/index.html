Choose the Value Your Backend Expects

Compare common synchronized values, then copy the JavaScript configuration that produces each format.

Scenario

A developer needs the visible tags and native form value to stay aligned while sending either tag1,tag2, tag1, tag2, or tag1 tag2 to an existing backend.

User goal: try each format and inspect the exact string that a form submission receives.

Try it

  1. Add or remove a tag in each editor and compare its synchronized value.
  2. In the first two editors, add tag 3 to confirm that an individual tag may contain spaces.
  3. In the strict space-delimited editor, try tag 3 and review the validation message.
  4. Activate Preview submitted values to inspect the three native form entries together.

JavaScript snippets

Default CSV: tag1,tag2

createTagInput(source);

Comma and space: tag1, tag2

createTagInput(source, {
  serializer: (tags) => tags.join(", ")
});

Strict space-delimited: tag1 tag2

createTagInput(source, {
  parser: (value) => value.trim().split(/\s+/),
  serializer: (tags) => tags.join(" "),
  invalidChars: /\s/,
  messages: {
    invalidChars: "Space-delimited tags cannot contain spaces."
  }
});

JSON for unambiguous multiword tags

createTagInput(source, {
  outputFormat: "json"
});

// ["tag1","tag 2"]

Developer notes

  • Import createTagInput from ../../dist/index.js and the package styles from ../../dist/styles.css.
  • Initialize a labelled input or textarea selected with [data-a11y-tag-input].
  • The source control remains the native form value unless valueElement or data-output-target selects another control.
  • separators controls tag entry; outputFormat and serializer control the synchronized value.
  • Use instance.setTags([...]) when application code needs to replace tags and synchronize the form value.

Accessibility notes

  • Each source textarea has a visible label and useful description before JavaScript runs.
  • Enter and configured separator keys add tags; Backspace from an empty field reaches the final remove button.
  • The plugin announces add, remove, duplicate, and validation feedback while the continuously changing value previews remain non-live.
  • The submitted preview is announced only after the explicit submit action, avoiding duplicate speech during editing.
  • Reset uses the native form reset behavior and does not intentionally move focus or add an announcement.
  • The shared demo styles provide responsive layout, visible focus, reduced-motion handling, and forced-colors borders.

Limitations

A space-delimited string cannot represent multiword tags without an escaping convention. Use JSON or another unambiguous serializer when tag values may contain spaces, commas, or other delimiter characters. This page previews submission locally and does not send data to a server.
