Preset Tag Sets For Support Triage

Support leads can apply common label bundles, then edit the final tag list before saving a ticket view.

Scenario

User goal: start from a useful preset without losing the ability to add or remove individual tags.

Try it

  • Choose the Escalation preset.
  • Add a custom tag such as refund.
  • Remove one preset tag with the keyboard.
  • Apply the Reset queue preset and confirm the form value changes.

Developer notes

Presets can be simple buttons that call setTags() on the instance. 

const presets = {
  escalation: ["urgent", "blocked", "manager-review"]
};

button.addEventListener("click", () => {
  instance.setTags(presets.escalation);
});

How to build it

Add native buttons for the preset choices, then use the instance API to replace the selected tags and return focus to the field.

HTML

<textarea id="ticket-tags"></textarea>
<button type="button" data-preset="escalation">
  Escalation
</button>

JavaScript

const instance = createTagInput(source);

button.addEventListener("click", () => {
  instance.setTags(["urgent", "blocked", "manager-review"]);
  instance.focus();
});

Accessibility notes

  • Preset controls are real buttons with visible text.
  • The status output announces which preset was applied.
  • The tag input keeps its normal keyboard add, remove, and focus behavior after a preset change.
  • Preset names are examples; use labels that match your workflow.

Limitations

  • This recipe demonstrates addon-style behavior around the core plugin; production forms still need target browser, assistive technology, touch, zoom, and forced-colors checks.
  • Keep labels, validation copy, and recovery instructions aligned with your own domain language before shipping.
