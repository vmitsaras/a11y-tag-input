Internationalization stress test

Tags Without Language Boundaries

A localization team can probe real scripts, input methods, emoji sequences, and hostile content lengths in one focused lab.

Scenario

User goal: verify that tag creation, reading, removal, wrapping, and focus behavior remain usable across varied writing systems and input methods.

Try it

  • With a Japanese or Chinese IME, compose the complete candidate, finish composition, then press Enter.
  • Pressing Enter, comma, or semicolon during active composition must not create a tag.
  • Navigate to and remove the Arabic, Hebrew, and emoji tags.
  • Test at 320 CSS pixels, 200% zoom, and in forced colors; the page should not gain horizontal scrolling.

Example

Compose IME text fully, end composition, then press Enter. Existing samples intentionally mix scripts and lengths.

How to build it

No special initialization is required. The core input already ignores delimiter and Enter keydown events while composition is active. Newline serialization keeps the authored samples readable.

createTagInput(source, { outputFormat: "newline" });

Accessibility notes

  • IME guidance is visible and programmatically associated with the field.
  • Sample values use automatic direction and plaintext bidi isolation where they are displayed outside the widget.
  • Tag text and remove-button names retain the complete authored value.
  • Long content wraps instead of being visually truncated.

Limitations

This page is a browser and assistive-technology test aid, not proof of universal language, font, speech, or IME support. Test the combinations required by your product.
